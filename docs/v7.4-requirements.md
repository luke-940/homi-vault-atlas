# Homi Vault Atlas v7.4 Requirements

This document is the public-safe implementation contract for the v7.4 Living Knowledge Terrain reform. It contains no private source locations, source hashes, governance identifiers, private document titles, or owner-only measurements.

## Product standard

Homi Vault Atlas is a commercial-quality knowledge product, not a prototype or operational console. v7.4 must make the represented knowledge understandable, navigable, honest about coverage, visually memorable, and safe to publish.

- Visual direction: **Living Terrain + bounded organic pulse**.
- Product Chrome, controls, browser title, and social title: English.
- Explanations, interpretation, and evidence: Korean-first with explicit language semantics.
- Theme: complete light theme only.
- Motion: finite, data-bound, and explanatory; no infinite loops or decorative particles.
- Knowledge remains the protagonist. Agency appears as provenance and navigation, never as document or relation counts.
- Historical review measurements are comparison evidence only. Every release metric is computed from the fresh capture.

## Profile boundary

One codebase produces two physically separated profiles.

### `atlas-owner`

- Local-only.
- May show the full safe structural inventory, measured aggregates, and privacy-safe activity summaries.
- Must not enter version control, hosted artifacts, release assets, or public search bytes.
- Must not expose raw governance events, source paths, receipts, secrets, or private document bodies.

### `atlas-public`

- The only deployable profile.
- Uses approved public titles or structural aliases.
- Contains aggregate relations, coverage explanations, and public-safe sources only.
- Must contain no owner-only byte sequence in JSON, JavaScript, HTML, DOM, Search, or ARIA text.

## Data contracts

### `atlas.inventory.v1`

The inventory records profile, capture date, physical Markdown count, named count, aggregate count, excluded count, exclusion reasons, folder coverage, unclassified count, and reconciliation status.

Required invariants:

- Every physical document is classified exactly once as `named`, `aggregate`, or `excluded`.
- `physical = named + aggregate + excluded`.
- `unclassified = 0` is a release gate.
- Exclusion precedence is deterministic: archive, scaffolding, control-internal, raw-daily, explicit-policy, public-name-not-approved.
- Hidden graph state is never treated as public-name permission.

### `atlas.structure.v2`

The structure supports districts, curated hubs, paper gateways, strategy insights and requests, projects and stages, signal domains and storylines, safe source documents, and aggregate boundaries.

Required invariants:

- Each document has one primary parent.
- Multiple hub or category memberships are association edges and never inflate document totals.
- Owner and public title policies are explicit.
- Public titles must be allowlisted; otherwise the projection uses a structural alias or aggregate.
- Active paper records, gateway records, and overlapping categories remain separate dimensions.

### `atlas.activity.v1`

This owner-only contract contains privacy-safe aggregates by role, unit kind, status, and date. It may support lifecycle visualization but must exclude raw events, internal identifiers, paths, receipts, and live status.

### Existing agency contract

The existing agency contract retains its role meaning and authority boundaries. Actor nodes never become knowledge documents, relation counts, matrix members, or evidence of live activity.

## Metric semantics

- Primary gravity: `uniqueInboundDocuments`.
- Secondary count: `inboundLinkOccurrences`.
- Link occurrences, unique document pairs, and typed evidence are separate units and are never added together.
- Recency uses meaningful frontmatter dates or date-shaped Daily and Weekly locations.
- File modification time is not semantic recency.
- A missing date is `unknown`, never zero, inactive, or retired.
- All UI labels, sort orders, legends, and tests must state the same unit.

## Product experience

### Home

Home leads with the real knowledge terrain rather than an agency diagram.

- Districts and important hubs form the primary terrain.
- Hub gravity controls prominence; important hubs may be materially larger and brighter than ordinary nodes.
- Only verified recent activity may produce a bounded finite pulse.
- Agency remains a concise provenance band with a distinct visual grammar.
- Scenes: `Living Terrain`, `Knowledge Gravity`, `Verified Activity Pulse`, `Coverage Boundary`.
- Home provides body-level entry points to Explore, Observe, Flow, Time, Agency, and Search.
- The public profile always explains named, aggregated, and excluded coverage.

### Explore

- Three levels: district → hub → profile-safe source.
- Owner shows the complete safe hub and project structure.
- Public shows approved names, structural aliases, and aggregates.
- Combine a top-ranked terrain with sorted or virtualized lists; never force the full inventory into one fixed canvas.
- Every deep node is reachable by keyboard and touch.

### Observe

- `Global Relations` keeps the district matrix.
- `Hub Relations` shows the selected hub's ranked or ego relationships.
- A single available relation layer is a static label, not a fake tab.
- Matrix labels state direction semantics and use pre-indexed lookup.

### Flow

- Only paths with real members are interactive.
- Zero-member paths and generated boundary labels are prohibited.
- If no public-safe flow exists, explain why with an explicit empty state.

### Time

- Show only events with real date or lifecycle evidence.
- Generated placeholder chronology is a build failure.
- Missing evidence is `unknown`, not zero or disappearance.
- If the public profile has no allowed chronology, show an honest empty state.

### Agency

- Preserve roles, responsibility, evidence return, and independent ownership boundaries.
- Provide links into relevant knowledge areas without mixing actor nodes into knowledge metrics.

### Search

- Separate `Operating Roles` from `Knowledge`.
- Search only what the active profile is allowed to display.
- Collapse repeated district or group labels.
- Actor results route to Agency; knowledge results route to the relevant knowledge workspace.

## Information architecture

A single `workspaceSceneRegistry` owns scene IDs, defaults, URL aliases, breadcrumbs, document titles, scene rails, and QA coverage.

- Every registered scene creates a distinct DOM or state.
- Legacy URLs recover through aliases with a clear status message.
- Semantic navigation history stores at most twenty meaningful states.
- Breadcrumbs read `Home → Workspace → Scene → Selection`.
- Browser titles read `Workspace · Scene · Homi Vault Atlas`.
- The brand control is the one Home item and participates in the same roving keyboard navigation without adding a duplicate Home tab.
- Malformed URLs recover to Home, not a deep workspace.

## Visual system

- Fixed district colors plus one Homi amber accent for current selection or verified activity.
- Numeric magnitude changes size and luminance within a district color.
- Shape, label, and line style duplicate critical color meaning.
- Three surface levels, four radius levels, three elevation levels, and four font weights.
- Minimum essential label size: 12px. Minimum body size: 14px.
- Korean explanatory text uses `keep-all` with safe overflow wrapping.
- The blueprint grid is substantially reduced; restrained celestial depth may support the terrain.
- Selection, hover, focus, and press states share one interaction vocabulary.

## Motion system

- First entry: at most 800ms and once per session.
- Scene transition: 420–560ms.
- Hover, press, and selection feedback: 120–180ms.
- Allowed properties: transform, opacity, and verified relation-line trace.
- Infinite animation count: zero.
- Hidden tabs pause motion.
- Reduced motion removes translation, zoom, and line drawing and uses immediate changes or a short fade.
- The static frame must communicate every relationship and boundary without relying on motion.

## Responsive system

- At least 1280px: full split layout.
- 1180–1279px: compact split with inspector overlay.
- 821–1179px: vertical cards with page scroll.
- At most 820px: dedicated mobile sibling.
- At short heights, compress Chrome and spacing but never clip content.
- Home stacks below 1180px.
- Time leaves its three-column layout below 1280px.
- Page-level horizontal overflow is always zero.

## Privacy and publication gates

Public bytes must reject private paths, private titles, governance identifiers, raw receipts, source hashes, ports, process identifiers, emails, phone numbers, IP addresses, tokens, secrets, live work state, command history, and model or session details.

- JSON is the audited source of truth.
- File-compatible JavaScript wrappers are generated from exact JSON bytes and verified deeply equivalent.
- Missing or stale public output is a failure.
- Owner bytes appearing in tracked files, public output, hosted artifacts, or release assets is a stop condition.
- The public coverage ledger may disclose safe aggregate counts and reason categories, never private source identity.

## QA and completion

- Inventory, folder, profile, and category totals reconcile.
- Public sources are a strict subset of owner sources.
- Every scene has real distinct state; ghost scenes equal zero.
- Placeholder patterns and clickable zero-member routes equal zero.
- Console warnings and errors equal zero.
- Label collisions, clipped labels, essential text below 12px, and page horizontal overflow equal zero at every required viewport.
- Golden screenshots and independent geometry checks both pass; a screenshot file alone is never proof.
- Keyboard-only, touch, Escape, focus restoration, normal motion, reduced motion, first visit, revisit, file URL, local HTTP, and hosted output are exercised.
- Local checks stay serial and resource-safe. The complete matrix runs once in pull-request CI and the verified artifact is reused for deployment.
- Publication stops on privacy leakage, unexplained coverage, reconciliation mismatch, torn capture, false operational claims, cross-profile leakage, or production readback failure.
